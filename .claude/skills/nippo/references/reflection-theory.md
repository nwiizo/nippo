# リフレクション理論まとめ

日報の問い生成時にClaude が参照する理論的背景。
問いのバリエーションを日ごとに変えるためにも活用する。

---

## コルブの経験学習サイクル（Kolb, 1984）

経験を学びに変える4段階のサイクル。日報の基本構造に対応する。

```
① 具体的経験（Concrete Experience）
   → 今日何が起きたか。事実の記録。

② 省察的観察（Reflective Observation）
   → それをどう感じ、何に気づいたか。
   → 問いの例:
     - この作業中、いちばん集中できた瞬間はいつでしたか？
     - 予想と違う結果になった場面はありましたか？
     - 手が止まったとき、最初に何を考えましたか？

③ 抽象的概念化（Abstract Conceptualization）
   → なぜそうなったか。一般化できる法則は何か。
   → 問いの例:
     - この経験から、他の場面でも使えそうなルールは？
     - 同じ詰まり方を前にもしたことがありますか？共通点は？
     - うまくいった進め方に再現可能なパターンは？

④ 能動的実験（Active Experimentation）
   → 明日どう試すか。
   → 問いの例:
     - 明日、ひとつだけ変えるとしたら？
     - それを試したら何が起きると予想しますか？
```

**核心**: 毎日このループを回すことで、経験が単なる思い出ではなく成長の糧になる。

## ギブスのリフレクティブサイクル（Gibbs, 1988）

コルブモデルの弱点（感情プロセスの欠如）を補う理論。
「どう感じたか」を省察の正式な要素として組み込む。

6段階:
1. Description（何が起きたか）
2. **Feelings**（何を感じたか）← ここが核心
3. Evaluation（何がうまくいき、何がうまくいかなかったか）
4. Analysis（なぜそうなったか）
5. Conclusion（何を学んだか）
6. Action Plan（次に何をするか）

**nippo での活用**: ①省察的観察の問いに感情を必ず含める。

```
技術的な事実だけでなく、感情も新鮮なうちに記録する。

良い問いの例:
  「rayon の lifetime エラーが解決したとき、何を感じましたか？
   3時間の試行錯誤の中で、感情はどう変化しましたか？」

避けるべき問い:
  「今日の作業で感じたことを書いてください」
  （漠然としていて深い内省につながらない）
```

## ALACT モデル（Korthagen, 2001）

教師教育の文脈で開発された、行為の奥にある思考・感情・意図まで掘り下げるモデル。
insight（週次・月次）の構造に使用する。

5局面:
1. **Action**（行為）
2. **Looking back on the action**（行為の振り返り）
3. **Awareness of essential aspects**（本質的な側面への気づき）
4. **Creating alternative methods of action**（代替手法の創出）
5. **Trial**（試行）

### 本質への気づきのための8つの問い

| 自分について | 相手について |
|------------|------------|
| 何をしたか？ | 相手は何をしたか？ |
| 何を考えていたか？ | 相手は何を考えていたか？ |
| 何を感じていたか？ | 相手は何を感じていたか？ |
| 何を望んでいたか？ | 相手は何を望んでいたか？ |

**nippo での活用**: 「相手」を Claude（AIエージェント）と読み替える。

```
- 自分は Claude に何を望んでいたか？
- Claude の応答のどこにギャップを感じたか？
- そのギャップは自分の指示の曖昧さから来ているか、
  それとも Claude の理解の限界から来ているか？
```

## デューイの経験の連続性（Dewey, 1938）

「すべての経験は、先行する経験から何かを受け取り、
後続する経験の質を何らかの形で変える」

**nippo での活用**: 日報の蓄積自体がリフレクション能力を育てる設計思想の根拠。
1日1日の問いへの回答が、翌日の行動選択に影響する。

## 問い生成時の使い分け

| 状況 | 使う理論 | 問いの特徴 |
|------|---------|-----------|
| 日報のリフレクション | コルブ + ギブス | 具体的場面に基づく、感情を含む |
| 週次・月次 insight | ALACT | 期間全体の傾向、本質への掘り下げ |
| コミュニケーション振り返り | ギブス + ALACT | 意図と伝達のギャップ |

### 問いの質を上げるポイント

1. **具体性**: 「今日」ではなく「14:30にrayon のエラーと格闘していたとき」
2. **感情**: 「何を思ったか」ではなく「何を感じたか」
3. **対比**: 「うまくいった場面とそうでない場面の違いは？」
4. **仮説**: 「もし〜していたら？」で別の可能性を考えさせる
5. **小ささ**: 「今後どうするか」ではなく「明日ひとつだけ変えるなら？」
